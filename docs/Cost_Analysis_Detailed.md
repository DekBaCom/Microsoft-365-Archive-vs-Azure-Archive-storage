# การคำนวณต้นทุนโดยละเอียด
## Microsoft 365 Archive vs Azure Archive Storage

---

## 📊 Scenario 1: เก็บไฟล์ 1 TB เป็นเวลา 3 ปี (ไม่มีการดึง)

### Microsoft 365 Archive

| รายการ | ค่า | หมายหนึ่ง |
|--------|-----|---------|
| ขนาดไฟล์ | 1,024 GB | = 1 TB |
| ราคาจัดเก็บ/เดือน | $0.05/GB | ต่อไฟล์เดือน |
| ค่าจัดเก็บเดือน | $51.20 | 1,024 × 0.05 |
| ค่า 3 ปี (36 เดือน) | $1,843.20 | 51.20 × 36 |
| ค่า Retrieval | $0 | ฟรีตั้งแต่มี.ค.2025 |
| ค่า Operations | $0 | รวมอยู่แล้ว |
| **รวมทั้งสิ้น** | **$1,843** | ไม่รวม quota |

#### หมายเหตุ:
- ✅ **ถ้าอยู่ในโควต้า**: ราคา = **$0**
  - SharePoint: 1 TB + 10 GB/ผู้ใช้
  - OneDrive: 1 TB/ผู้ใช้
- ⚠️ **ถ้าเกินโควต้า**: ต้องเสียค่า $0.05/GB/เดือน

---

### Azure Archive Storage

| รายการ | ค่า | หมายหนึ่ง |
|--------|-----|---------|
| ขนาดไฟล์ | 1,024 GB | = 1 TB |
| ราคาจัดเก็บ/เดือน | $0.00150/GB | ต่อไฟล์เดือน |
| ค่าจัดเก็บเดือน | $1.54 | 1,024 × 0.00150 |
| ค่า 3 ปี (36 เดือน) | $55.36 | 1.54 × 36 |
| ค่า Write Operations | $0.10-0.20 | การอัพโหลดครั้งแรก |
| ค่า Read Operations | $0 | (ไม่มีการดึง) |
| ค่า Retrieval | $0 | (ไม่มีการดึง) |
| **รวมทั้งสิ้น** | **~$55-75** | ราคาต่ำสุด |

#### หมายเหตุ:
- ⚠️ **ปัญหาไฟล์เล็ก**: ตั้งแต่ก.ค.2026 เสียค่า 128 KiB ขั้นต่ำต่อไฟล์
  - หากไฟล์ 10,000 ไฟล์ @ 50 KB = เสียค่า 10,000 × 128 KB
  - เท่ากับ 1.28 TB (แต่ข้อมูลจริง 500 GB)

## 🎯 Break-even Analysis

### เมื่อไหร่ที่ Azure ถูกกว่า M365?

**ปัจจัย**:
- M365: $0.05/GB/เดือน
- Azure: $0.00150/GB/เดือน (กล่าว: 33 เท่าถูกกว่า)

**สถานการณ์**:
```
M365 ค่า = Azure ค่า
$0.05 × GB × เดือน = ($0.00150 × GB × เดือน) + (retrieval + ops)

ข้อมูล 1 TB เก็บ 3 ปี:
- M365: $1,843
- Azure: $55
- ประหยัด: $1,788
```

**Azure ถูกกว่าเสมอ เว้นแต่**:
- ต้องดึงข้อมูลบ่อยมาก (10+ ครั้ง/เดือน)
- หรือไฟล์เล็กมาก (< 128 KB)

## 🔢 ตัวคำนวณเครื่องเอง

### Microsoft 365 Archive
```
ค่าจัดเก็บ/ปี = ขนาด (GB) × 12 × $0.05
ค่า 3 ปี = (ขนาด × 36 × $0.05)
ค่า Retrieval = $0 (ฟรี)
```

**ตัวอย่าง**: 500 GB
- ปีละ: 500 × 12 × 0.05 = $300
- 3 ปี: 500 × 36 × 0.05 = $900

### Azure Archive Storage
```
ค่าจัดเก็บ/ปี = ขนาด (GB) × 12 × $0.00150
ค่า Retrieval = ขนาด × จำนวนครั้ง × $0.02
ค่า Ops = จำนวนครั้ง × $0.55/10K
```

**ตัวอย่าง**: 500 GB ดึง 2 ครั้ง/ปี
- จัดเก็บ 3 ปี: 500 × 36 × 0.00150 = $27
- Retrieval: 500 × 6 × 0.02 = $60
- Ops: 6 × 0.000055 = $0.33
- รวม: ~$87

## 💡 สำคัญ: Minimum Billable Size (ตั้งแต่ก.ค.2026)

Azure Archive จะคิดค่า **128 KiB ขั้นต่ำต่อไฟล์**

| ขนาดไฟล์จริง | ขนาดที่คิดค่า | ตัวอย่าง |
|----------|------------|--------|
| 1 KB | 128 KB | 128 × |
| 4 KB | 128 KB | 32 × |
| 50 KB | 128 KB | 2.56 × |
| 128 KB | 128 KB | 1 × |
| 1 MB | 1 MB | 1 × |

### ตัวอย่าง: 10,000 ไฟล์ @ 10 KB

**ข้อมูลจริง**: 
- 10,000 × 10 KB = 100 MB = 0.1 GB

**คิดค่าต่อ**:
- 10,000 × 128 KB = 1,280 MB = 1.28 GB
- **เสียค่า 1.28 GB แต่ข้อมูลจริง 0.1 GB** ❌

## 🎯 คำแนะนำการเลือก (ตามต้นทุน)

### **ถูกสุด: Azure Archive**
```
ต้องการ:
✓ ต้นทุนต่ำสุด
✓ ไฟล์ขนาดใหญ่
✓ ไม่ต้องดึงบ่อย
✓ เก็บ 180+ วัน

ราคา: ~$2-20/TB/ปี
```

### **สมดุล: Microsoft 365 Archive**
```
ต้องการ:
✓ เข้าถึงไว
✓ ค้นหาได้ (eDiscovery)
✓ ไฟล์จาก M365
✓ ไม่คิดค่าเพิ่มเติม (ถ้ามีโควต้า)

ราคา: ~$0-612/TB/ปี
```

### **Hybrid: ทั้ง 2 แบบ**
```
ต้องการ:
✓ ข้อมูล M365 → M365 Archive
✓ ข้อมูลอื่น → Azure Archive
✓ ใช้ประโยชน์สูงสุด

ราคา: ขึ้นอยู่กับอัตราส่วน
```
