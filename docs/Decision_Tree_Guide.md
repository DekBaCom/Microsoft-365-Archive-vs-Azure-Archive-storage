## 🎯 Path 1: ไฟล์มาจาก Microsoft 365

```
START: ไฟล์ใน SharePoint/OneDrive/Teams
│
└─ คำถาม: เข้าถึงบ่อยหรือไม่?
   │
   ├─ YES (ทุกเดือน) → ❌ อย่าใช้ Archive
   │                   ✅ ใช้ Normal Storage
   │
   └─ NO (ไม่บ่อย) → ✅ Microsoft 365 Archive
      │
      └─ คำถาม: ต้อง eDiscovery ไหม?
         │
         ├─ YES → 🎯 MUST USE M365 Archive
         │        เสียค่า แต่ได้ compliance
         │
         └─ NO → M365 Archive ยังดี
                 ✓ เข้าถึงไว
                 ✓ ไม่มีค่า retrieval
                 ✓ ใช้ quota (อาจฟรี)
```

### 💰 ต้นทุน M365 Archive Path
```
ค่าจัดเก็บ: $0.05/GB/เดือน
ค่า Retrieval: $0 (ฟรี)
3 ปี: 1 TB = $1,843 (หรือ $0 ถ้าในโควต้า)
```

## 🎯 Path 2: ไฟล์มาจาก Azure/อื่น ๆ

```
START: ไฟล์ใน Azure Blob/S3/อื่น ๆ
│
└─ คำถาม: เข้าถึงบ่อยหรือไม่?
   │
   ├─ YES (ทุกสัปดาห์) → ❌ อย่าใช้ Archive
   │                     ✅ ใช้ Hot/Cool
   │
   └─ NO (ไม่บ่อย) → ✅ Azure Archive
      │
      └─ คำถาม: ต้อง eDiscovery ไหม?
         │
         ├─ YES → ❌ Azure Archive ไม่ได้
         │        ✅ ย้ายไปใน M365 ก่อน
         │
         └─ NO → Azure Archive ดีที่สุด
                 ├─ ต้นทุนต่ำ: $0.001-0.002/GB/เดือน
                 ├─ 3 ปี 1 TB: ~$55 (ประหยัด 97%!)
                 └─ เวลาดึง: 15 ชม.
```

### 💰 ต้นทุน Azure Archive Path
```
ค่าจัดเก็บ: $0.00150/GB/เดือน
ค่า Retrieval: $0.02/GB (ถ้าต้องดึง)
3 ปี ไม่ดึง: 1 TB = $55
3 ปี ดึง 1 ครั้ง: 1 TB = $75
```

## 📋 Decision Matrix

### ตัวอักษร: คะแนน 1-5 (5 = ดีที่สุด)

| เกณฑ์ | M365 Archive | Azure Archive | ผู้ชนะ |
|------|-------------|--------------|--------|
| **ราคา** | 2 | 5 | Azure ✅ |
| **เข้าถึงไว** | 5 | 1 | M365 ✅ |
| **Compliance/eDiscovery** | 5 | 1 | M365 ✅ |
| **ยืดหยุ่น** | 2 | 5 | Azure ✅ |
| **ไม่มีค่า Retrieval** | 5 | 2 | M365 ✅ |
| **ไม่ต้อง M365 License** | 1 | 5 | Azure ✅ |

**วิธีใช้**:
1. ตรวจสอบว่าเกณฑ์ไหนสำคัญสำหรับคุณ
2. ดูผู้ชนะในเกณฑ์ที่สำคัญ
3. ตัดสินใจตามความสำคัญ

## 🔍 ตัวอย่างกรณีศึกษา

### Case 1: ห้องเก็บเอกสารทีมกฎหมาย
```
✓ ไฟล์ใน SharePoint
✓ ต้องการ eDiscovery สำหรับคดี
✓ ไม่เข้าถึงบ่อย (1-2 ครั้ง/ปี)
✓ Compliance สำคัญ

🎯 เลือก: Microsoft 365 Archive
💰 ราคา: ~$1,843/ปี (1 TB) แต่จำเป็น

สาเหตุ:
- ต้อง eDiscovery = M365 Archive เท่านั้น
- ราคาแพง แต่ได้ compliance
```

### Case 2: Data Lake ของบริษัท
```
✓ ไฟล์ใน Azure Blob Storage
✓ ข้อมูลวิเคราะห์แบบ Historical
✓ ไม่ต้อง eDiscovery
✓ ต้องการต้นทุนต่ำ

🎯 เลือก: Azure Archive
💰 ราคา: ~$55/ปี (1 TB) ประหยัด 97%!

สาเหตุ:
- ราคาต่ำสุด
- ไม่ต้อง Compliance
- Lifecycle Policies ง่าย
```

### Case 3: บ็อกเกจเลกาซี่
```
✓ ไฟล์มาจาก System เก่า (SQL Server backups)
✓ ต้องเก็บตามกฎหมาย 7 ปี
✓ ต้องดึง 1 ครั้งต่อปี (เพื่อตรวจสอบ)
✓ ไม่ใช่ M365 data

🎯 เลือก: Azure Archive + Lifecycle
💰 ราคา: ~$200-300/ปี (1 TB)

สาเหตุ:
- ประหยัดกว่า M365
- ไม่ต้อง compliance จริง ๆ
- Lifecycle Policy: Hot → Cool → Archive
```

## 📝 Quick Reference Cards

### ✅ เลือก M365 Archive ถ้า:
```
□ ไฟล์ใน SharePoint/OneDrive/Teams
□ ต้อง eDiscovery/DLP/Compliance
□ เข้าถึงอย่างไม่ค่อยบ่อย (ไม่ทุกวัน)
□ มี M365 license อยู่แล้ว
□ ต้องการเข้าถึงข้อมูลไว ๆ

ราคา: $0-612/TB/ปี
เวลาดึง: ทันที
ความเสี่ยง: ราคาแพง
```

### ✅ เลือก Azure Archive ถ้า:
```
□ ไฟล์ใน Azure Blob/S3/เซิร์ฟเวอร์
□ ไม่ต้อง eDiscovery/Compliance
□ เข้าถึงน้อยมาก (< 1 ครั้ง/เดือน)
□ ต้องการต้นทุนต่ำสุด
□ ไฟล์ขนาดใหญ่ (> 128 KB)

ราคา: $2-20/TB/ปี
เวลาดึง: 15 ชม.
ความเสี่ยง: ดึงข้อมูลช้า
```

## 🚀 Implementation Steps

### ถ้าเลือก M365 Archive:
```
1️⃣  ตรวจสอบ Quota
    - SharePoint: 1 TB + 10 GB/ผู้ใช้
    - OneDrive: 1 TB/ผู้ใช้

2️⃣  ตั้ง Lifecycle Policy
    - Archive sites หลังจาก 2-3 ปี
    - หรือตั้ง file-level (เมื่อ GA)

3️⃣  ทดสอบ eDiscovery
    - ตรวจสอบว่าค้นหาข้อมูลจัดเก็บได้

4️⃣  ตรวจสอบค่าจริง
    - ติดตามค่าที่เสียจริง
```

### ถ้าเลือก Azure Archive:
```
1️⃣  ตั้ง Lifecycle Policies
    - Hot → Cool (30 วัน)
    - Cool → Cold (90 วัน)
    - Cold → Archive (180+ วัน)

2️⃣  ตรวจสอบขนาดไฟล์
    - ตั้งแต่ก.ค.2026: ขั้นต่ำ 128 KB/ไฟล์

3️⃣  วางแผนการดึงข้อมูล
    - Rehydration: 15 ชม.
    - Retrieval fee: $0.02/GB
    - Ops: $0.55/10K reads

4️⃣  ติดตามต้นทุน
    - ใช้ Azure Cost Management
    - Alert ถ้าดึงข้อมูลบ่อย
```

## ⚠️ Common Mistakes

### ❌ ผิด 1: เลือก Azure Archive สำหรับข้อมูล M365
```
ปัญหา: ไม่สามารถค้นหา eDiscovery ได้
แก้ไข: ย้ายไป M365 Archive หรือเก็บไว้ใน M365
```

### ❌ ผิด 2: ใช้ Azure Archive สำหรับไฟล์เล็ก
```
ปัญหา: เสียค่า 128 KiB ขั้นต่ำต่อไฟล์
แก้ไข: รวมไฟล์เล็กเป็น ZIP หรือใช้ Cool Storage
```

### ❌ ผิด 3: ไม่คำนวณ Retrieval Fee
```
ปัญหา: ค่าดึงข้อมูลแพง
แก้ไข: คำนวณอัตราการเข้าถึง ต้อง

จริง ๆ
```

### ❌ ผิด 4: ลืมตรวจสอบ Quota
```
ปัญหา: M365 Archive ไม่ฟรี แต่คิดว่าฟรี
แก้ไข: ตรวจสอบ SharePoint quota เสมอ
```
